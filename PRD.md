# Learn-in-Public Streak Tracker

Build a app (web) that lets someone track what they're learning and share their progress with a public link. This mirrors what Team Shiksha is all about — learning in public — and touches the same kind of work you'd do on our real projects (RSVP, Openlogo).

Keep it simple, but make it yours. The interesting decisions are left open on purpose. We care about how you build and reason, not how big it is.

# The Problem

- A user lands on the app and enters what they're learning (e.g. "System Design").
  - user has to login first and in the dashboard they can creat multiple paths/learning/...
  - for creating a path it ask what they want to learns
- They get a shareable link (a unique slug — no login/email required to view).
  - this path will be accessable over the internet via a /slug
  - analytics for the path page (future)
  - user can add learning to the path and sub learning and add note kind of comment on that
  - below the whole path/log there will be the stats of the that path like users streak, heatmap of there logs
- This problem statement is open-ended, and so show us your creativity.
  - user can add x account to there profile, use 8bit avatars,
  - leaderboard that have highest streak
  - best learning day, longest streak
  - daily changing quotes on different pages
  - milestone celebration like badge on completing a streak bla bla
  - share progress card
  - user can have multiple paths

# Requirements

Must have

- A frontend — framework of your choice.
- A backend / API layer — your choice of language/framework.
- A real datastore of your choice (SQL or NoSQL). No in-memory/JSON-file fakes.
- Shareable link that renders a specific user's public log.
- Daily logging + a streak or heatmap view.
- Deployed live (Vercel / Railway / anything public) with a working URL.
- A README explaining your choices (see below).

# Submission

- Push your code to a public GitHub repo.
- Include your live deployed URL in the README.
- Record a 2–3 minute Loom (or any screen recording) where you:
  walk through one non-trivial decision you made and why, and
  show one bug you hit and how you fixed it.
- Share the repo link + Loom link as instructed in the application.
- The Loom or any screen recording is required. It's the fastest way for us to understand your thinking.

Note: If you have any other doubts, please ask for help in TeamShikshah Help Channel, tagging Tejas(@tejask3) or Anuj(@aightbitbyte).

We're not looking for a finished product. We're looking for someone who builds carefully, thinks about edge cases, and can explain their work. Have fun with it.
